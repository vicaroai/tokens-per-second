# tokens-per-second

Weekly, reproducible **tokens-per-second** benchmarks across LLM models and providers.

A GitHub Action runs every Monday (and on demand) against a predefined list of
models, measures streaming throughput with an identical prompt, and opens a pull
request that refreshes the leaderboard below. The numbers are generated by
machine, committed via reviewed PR, and backed by a documented methodology — so
they stay honest and comparable over time.

- **What's measured & how:** [docs/METHODOLOGY.md](docs/METHODOLOGY.md)
- **The model list:** [`benchmarks/models.yaml`](benchmarks/models.yaml)
- **Raw results (source of truth):** [`benchmarks/results/latest.json`](benchmarks/results/latest.json) · weekly snapshots in [`benchmarks/results/history/`](benchmarks/results/history/)

## How it works

1. [`benchmarks/models.yaml`](benchmarks/models.yaml) declares the providers and
   models to benchmark. It's edited by humans via pull request.
2. The [Benchmark workflow](.github/workflows/benchmark.yml) runs `tps bench`,
   which streams an identical completion from each model, records
   time-to-first-token and tokens/sec (median of several runs), and writes the
   results.
3. The workflow opens a PR updating the results files and this README. Once CI
   is green it auto-merges. The benchmark bot only ever writes data — never code.

## Running locally

```bash
cp .env.example .env   # fill in the provider keys you have
set -a; source .env; set +a

go run ./cmd/tps bench            # run the full benchmark, update results + README
go run ./cmd/tps render           # just re-render the README from latest.json
```

Missing a provider's key? That provider's models are recorded as failures and
skipped — the rest still run.

## Adding a model or provider

Open a PR editing [`benchmarks/models.yaml`](benchmarks/models.yaml). Adding a
model to an existing provider is a one-line change. A new provider that speaks
the OpenAI-compatible `/chat/completions` API needs only a config entry; a
provider with a different API needs a small adapter in
[`internal/provider/`](internal/provider/). See
[CONTRIBUTING.md](CONTRIBUTING.md).
