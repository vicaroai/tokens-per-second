## How it works

1. [`benchmarks/models.yaml`](benchmarks/models.yaml) declares the providers and
   models to benchmark (human-edited via PR).
2. The [Benchmark workflow](.github/workflows/benchmark.yml) runs weekly (and on
   demand), streaming an identical completion from each model and recording
   tokens/sec, time-to-first-token, and estimated cost (median of several runs).
3. It opens a PR that refreshes the results + the table above; once CI is green
   it auto-merges. The benchmark bot only ever writes result data — never code.

## Run it yourself

```bash
cp .env.example .env   # fill in the provider keys you have
set -a; source .env; set +a

go run ./cmd/tps bench    # run the full benchmark, update results + README
go run ./cmd/tps render   # just re-render the table from latest.json
```

Missing a provider's key? Those models are recorded as failures and skipped —
the rest still run.

## Add a model or provider

Open a PR editing [`benchmarks/models.yaml`](benchmarks/models.yaml). Adding a
model to an existing provider is one line. A provider that speaks the
OpenAI-compatible `/chat/completions` API needs only a config entry; one with a
different API needs a small adapter in
[`internal/provider/`](internal/provider/). See [CONTRIBUTING.md](CONTRIBUTING.md).
